Web Accessibility

What is Web Accessibility?

Web accessibility is the process of designing digital content in a way that accommodates the varying abilities of people with disabilities, with the goal of ensuring equal access and usability for all internet users.

The WAI Initiative

What is the Web Accessibility Initiative?

The World Wide Web Consortium’s (W3C) commitment to lead the web to its full potential includes promoting a high degree of usability for people with disabilities. The Web Accessibility Initiative (WAI) is an initiative of the W3C.

What does the WAI Do?

The WAI carries out many things such as ensuring the standards support accessibility, developing accessibility guidelines, supporting education on the matter of accessibility, and coordinating research and development relating to accessibility.

The 4 Tenets of Web Accessibility

Perceivable by at least one sense

Operable, using keyboard, mouse, voice...

Understandable, it should make sense to the users

Robust, content should use web standards compatible with different browsers, currently and in the foreseeable future

Web Content Accessibility Guidelines

WCAG 3 is the upcoming version of Web Content Accessibility Guidelines by WAI. It's being developed with updated guidelines and success criteria to improve web content accessibility for people with disabilities. WCAG 3 aims to make emerging technologies, such as mobile and wearable devices, more accessible to users with disabilities.

The Importance of Semantic Tags

Semantic tags are text identifiers that describe the meaning and purpose of data or elements. They can be used to apply a common understanding across different data sources, to simplify the calculation of measures and KPIs for reports, to help screen readers and code optimization, and to introduce new elements in HTML.

Using ARIA in Webpages

Aria is attributes you can add to HTML elements that make them more accessibile to users with disabilities who utilize assistive technologies such as screen readers.

ARIA has three components that are roles, states, and properties. Roles define a type of UI element. Different roles can be helpful for different purposes. For example, a landmark role provides easier navigation as it identifies each section of a webpage. States and properties are used to support ARIA roles.

The first rule is that you must use native HTML at all times unless it is for sure impossible you can't make it accessible otherwise. The second rule is to not change HTML semantics unless you have to. The third rule is that all interactive ARIA controls must be accessible. A final rule is that you must give all elements an accessible name.

You should use ARIA when needing to add more descriptive labels to buttons, for alerts, for relationships, and forms.
